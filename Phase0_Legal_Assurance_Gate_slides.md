# Phase 0: Legal & Assurance Gate
**Purpose:** Establish legal, compliance, and risk clarity before any technical work begins.
**Principle:** No build starts without signed legal assurances.

---

# Phase 0 Objectives
- Define **scope boundaries** and client responsibilities
- Confirm **data protection obligations**
- Verify **operator/third‑party compliance**
- Establish **liability & warranty limits**
- Approve **security controls**
- Confirm **IP ownership & transparency**

---

# Phase 0 Mandatory Deliverables
**All six must be approved before Phase 1:**
1) Scope & Responsibility Statement
2) Data Processing Annex (DPA)
3) Third‑Party Terms Compliance Review
4) Liability & Warranty Boundary Clauses
5) Security Controls Statement
6) IP & Ownership Addendum

---

# Deliverable 1: Scope & Responsibility Statement
**Defines what the system does and does not do**
- Monitoring + notification only
- Human confirmation required for booking
- Not a decision maker / not a visa authority
- User responsibility for final submission

**Outcome:** eliminates ambiguity and reduces legal exposure.

---

# Deliverable 2: Data Processing Annex (DPA)
**Defines GDPR/KVKK alignment**
- Lawful basis (consent / legitimate interest)
- Data minimization
- Encryption at rest + in transit
- Retention & deletion policy
- User rights (export / correction / deletion)
- Audit log requirements

---

# Deliverable 3: Third‑Party Terms Review
**Ensures operator compliance**
- Review terms of VFS, iDATA, BLS, Kosmos
- Confirm allowable data access actions
- Define polite request intervals
- Identify restrictions and ambiguity

---

# Deliverable 4: Liability & Warranty Boundaries
**Establishes risk containment**
- No guarantee of slot availability
- No liability for operator site changes/downtime
- Notification delays not guaranteed
- User responsible for final booking

---

# Deliverable 5: Security Controls Statement
**Minimum security baseline**
- Role‑based access
- Audit logging
- Credential handling policies
- Secure storage for sensitive data
- Incident response expectations

---

# Deliverable 6: IP & Ownership Addendum
**Confirms ownership & transparency**
- OmnisArx ownership of code + IP
- Full documentation and reproducibility
- No black‑box modules
- White‑label / scalability rights

---

# Phase 0 Exit Criteria (Go/No‑Go)
✅ All six deliverables approved
✅ Risk register accepted by OmnisArx
✅ Legal counsel sign‑off (if required)
➡ Move to Phase 1: Scope & Requirements

---

- **Legal Counsel (if engaged):** validates compliance
